# 编剧共同内核

本文件是女频版与男频版共用的编剧事实源。它规定从立项到剧本交接的共同工艺，不定义任何性别群体的固定心理，也不承诺作品成为爆款。

执行编剧任务时，先读本文件，再只读当前版本对应的故事引擎。用户明确要求跨频或混合类型时，才同时使用两个故事引擎，并说明采用了哪些主次观看承诺。

## 一、执行契约

### 1. 先读取项目配置

开始写作前，读取并记录下列字段。不要用固定集数、固定批次或默认性别关系覆盖用户配置。

| 字段 | 处理规则 |
|---|---|
| 版本 | 女频、男频或用户明确指定的混合版本 |
| 呈现形式 | 真人短剧、动态漫、横屏剧集、小说大纲或用户自定义形式 |
| 剧集模式 | 长剧或短剧；按用户选择执行，不根据集数自行替用户改判 |
| 是否分季 | 开启或关闭 |
| 总季数 | 允许留空；留空表示不预设上限，只规划当前范围 |
| 单季集数 | 允许留空；留空时根据当前故事目标提出建议，不视为固定值 |
| 全剧总集数 | 允许留空；仅记录用户已经确认的全系列上限 |
| 当前规划集数 | 记录本次开发或制作范围；不得自动等同于全剧总集数 |
| 单集时长 | 读取用户设置；没有设置时使用范围而非伪精确秒数 |
| 当前进度 | 当前阶段、当前季（如适用）、当前集、已确认内容和待写范围 |
| 题材与观看承诺 | 区分主类型、次类型、主要承诺和次要承诺 |
| 角色与世界规则 | 读取已确认的人物档案、能力边界、关系和世界规则 |
| 本次交付 | 诊断、人物档案、全剧大纲、分季大纲、分集大纲、完整剧本或改稿 |
| 批次大小 | 读取用户设置；留空时根据审校负担提出建议 |
| 内容与制作约束 | 预算、演员、场景、受众年龄、尺度、专业领域和禁止项 |

### 2. 区分关键缺口与可逆缺口

- 缺少会改变作品本质的信息时，先向用户提出一个最关键的问题。例如：用户同时要求纯悬疑和高甜恋爱，却没有说明主次。
- 缺少可逆的小参数时，继续工作并在交付开头列出采用的假设。例如：单集时长未定，可先按“约 2 至 3 分钟”规划。
- 用户已经明确说“直接做”“开工”或同义指令时，不另设阻塞式确认；做出最小、可逆且可见的假设。
- 不把平台、年龄或性别标签自动翻译成心理结论。以用户给出的观看承诺为准。

### 3. 维护单一事实源

将下列材料按优先级视为项目事实源：

1. 用户本轮明确确认的修改；
2. 最新项目档案或系列圣经；
3. 最新已确认剧本与连续性记录；
4. 已确认的大纲；
5. 早期草案和创作种子。

发现冲突时，列出冲突并请求裁决或采用最新明确指令。不要静默混合两个版本。

## 二、编剧与导演的职责边界

### 编剧负责

- 设计故事承诺、主题问题、人物目标和人物弧；
- 设计每集发生什么、为什么发生以及发生后改变什么；
- 写场景、行动、关键表演状态、对白和必要的内心独白；
- 管理伏笔、信息差、关系变化和连续性；
- 把文学剧本交给导演继续转化。

### 编剧默认不负责

- 逐秒拆分镜头；
- 指定景别、机位、焦段、运镜、灯光参数和剪辑点；
- 为特定生成工具编写生图或生视频提示词；
- 进行最终配音语速、镜头时长和制作排期核算；
- 代替选角、摄影、美术或声音工种做终局决策。

可在“给导演的提示”中写希望观众感受到的压迫、亲密、荒诞、失重或释然，但不要把情绪意图伪装成镜头指令。用户明确要求拍摄稿时，先完成可审校的文学稿，再进入导演工作流。

## 三、标准工作流

### 步骤 1：建立创作配置

记录版本、形式、长短剧、分季参数、本次范围、故事承诺和制作约束。

完成标准：不存在会让后续结构产生两种完全不同结果的未决问题；其余缺口已转化为显式假设。

### 步骤 2：做故事承诺诊断

诊断观众为什么继续看、主角必须改变什么、阶段兑现是什么、设定有哪些结构风险。不要给“爆款概率”或商业保证。

完成标准：主要承诺、次要承诺、核心问题和可验证的阶段兑现各有一句清晰表述。

### 步骤 3：搭建四层结构

先确定全剧；开启分季时再确定当前季；然后拆单集，最后写场景。不要从零散台词反推整个故事。

完成标准：当前交付范围内，每一层都有目标、阻力、转折和结果；开启分季时，当前季能独立兑现一个实质承诺。

### 步骤 4：建立人物与规则档案

为主要人物写目标、资源、边界、盲点、行为锚点和关系状态；为特殊能力、职业流程或世界机制写边界。

完成标准：每个主要人物都能主动选择；剧情不依赖角色临时降智。

### 步骤 5：写当前批次

每集先写戏剧任务和状态变化，再展开场景。确保局部兑现先于新的前向问题。

完成标准：每集至少改变一个故事状态；没有一集只重复争执或只停在“马上反击”。

### 步骤 6：执行硬门槛自查并修订

逐项检查共同门槛和当前版本门槛。发现未通过项时先改稿，再交付；不要把未修复的问题留给一张形式化清单。

完成标准：所有适用门槛通过，或明确标注用户要求保留的有意识例外及其风险。

### 步骤 7：交接并更新连续性

输出导演可接手的文学剧本，并更新人物、关系、资源、信息、风险和伏笔状态。

完成标准：下一位创作者不依赖聊天历史，也能从交付物继续工作。

## 四、故事承诺诊断

### 1. 用观看情境代替人群想象

先定义具体观看情境，再判断叙事策略。例如：

- 想快速获得一次清晰兑现；
- 想持续追踪人物成长；
- 想体验关系协商与情感修复；
- 想看复杂问题被专业能力解决；
- 想进入陌生世界并逐步理解规则；
- 想在家庭、团队或社区关系中获得共鸣。

不要写“女性都缺什么”或“男性天然喜欢什么”。“女频”和“男频”只表示本项目采用哪一组市场叙事惯例。

### 2. 诊断六个字段

- **目标观看情境**：观众在什么状态下打开作品，希望获得什么体验；
- **主要承诺**：作品最稳定、最应被兑现的体验；
- **次要承诺**：增强层次但不能抢走主线的体验；
- **核心问题**：主角必须回答的行动性问题；
- **阶段兑现**：当前剧集或当前规划范围（开启分季时可为当前季）结束时，什么必须真实改变；
- **差异化机制**：职业、规则、关系、空间、叙述方式或视觉意象中的独特变量。

### 3. 输出强项和风险，不输出保证

可判断：

- 承诺是否清楚；
- 主角是否主动；
- 冲突是否可持续；
- 兑现是否有因果；
- 体量是否匹配；
- 制作是否可行；
- 是否存在陈旧套路、刻板印象或专业失真。

不要把任何结构公式写成成功保证，也不要声称某平台或某群体必然偏好某种内容。用户请求当前市场判断时，应另做有日期、有来源的调研。

## 五、四层结构

### 1. 全剧层

回答：

- 谁要完成什么改变；
- 什么力量长期阻止他或她；
- 每次选择会付出什么；
- 终局前必须解决哪个核心矛盾；
- 结束时人物、关系、资源、秩序或世界发生什么不可逆变化。

### 2. 分季层

开启分季时，为当前季定义：

- 本季外部目标；
- 本季内部或关系目标；
- 本季主要对手或系统阻力；
- 本季中段的规则变化；
- 本季高潮中的决定性选择；
- 本季必须回收的承诺；
- 下一季接口。

总季数留空时，不虚构无限季大纲。只建立当前季闭环和可扩展接口。单季集数留空时，先按事件密度和制作条件提出区间，再由用户确认或允许动态规划。

### 3. 单集层

使用以下功能顺序，不强迫每集采用完全相同的节拍数量：

1. **开场状态或扰动**：尽快让目标、异常、风险或关系变化可见；
2. **本集即时目标**：主角这一集具体想完成什么；
3. **阻碍与行动回合**：对手、规则、资源或关系如何真实阻挡；
4. **选择与代价**：主角必须决定做什么并承担后果；
5. **局部兑现**：解决、揭示、赢得、失去、靠近或离开；
6. **状态变化**：至少一项故事资产发生变化；
7. **前向问题**：产生下一步目标、风险或未知。

结尾可以是悬念、关系变化、利益变化、新目标、余韵或主题回声，不要求每集用假危机截断。不要把观众已经应得的信息故意拖延成“下一集再说”。

### 4. 场景层

每个场景至少明确：

- 谁在场；
- 场景发起者想要什么；
- 谁或什么阻挡；
- 人物采取哪些可见行动；
- 信息、权力或情绪在哪一刻转向；
- 场景结束时，人物带着什么新状态离开。

删除既无信息、无关系变化、无行动结果，也不承担必要氛围功能的场景。

### 5. 结构比例是工具，不是定律

三幕、五幕、30/40/30、任务单元或循环结构都可以作为起始工具。根据体量、题材和角色弧调整，不把任何比例称为“情绪物理规律”。

## 六、人物引擎

### 1. 建立九项人物档案

为每个主要人物记录：

1. 当前身份与处境；
2. 外在目标；
3. 内在需求或尚未承认的真相；
4. 可用资源和真实能力；
5. 能力边界与脆弱点；
6. 道德、情感或生存边界；
7. 错误信念或盲点；
8. 行为与语言锚点；
9. 本阶段可能发生的变化。

身份标签不是人物。职业、财富、性别、年龄、MBTI 或“霸总”“甜妹”“战神”都必须被翻译成具体行为、选择和限制。

### 2. 用行为锚点保持一致

行为锚点应可观察，例如：

- 面对压力先收集证据，还是先保护同伴；
- 生气时变得更简短，还是说得更快；
- 习惯正面谈判，还是通过交换资源解决；
- 在谁面前会破例，破例需要什么触发；
- 什么底线即使失败也不跨越。

人物改变时，写出触发、抵抗、试错、选择和后果。不要为了完成反转让人物突然换一套价值观。

### 3. 谨慎使用人格类型

用户主动提供 MBTI 或其他人格类型时，只将其作为非诊断性的创作速记。优先相信项目中的行为证据。不要让人格标签决定台词长短、职业能力、道德水平或亲密关系模式。

### 4. 让配角拥有主体性

主要配角至少拥有自己的目标、资源、边界和一次独立选择。配角可以服务主线，但不能只负责：

- 解释设定；
- 无条件吹捧主角；
- 被主角收集或拯救；
- 机械制造误会；
- 作为性别、阶层、疾病或职业刻板印象。

## 七、因果链与状态变化

### 1. 使用转折因果句

每个重要转折都能写成：

> 因为人物掌握了某项信息或承受了某种压力，所以主动采取某项行动；行动造成具体后果，并迫使自己或他人做出下一次选择。

如果无法写出这句话，转折通常依赖作者强推、巧合或角色降智。

### 2. 巧合只制造问题

巧合可以让两个人相遇、让证据暂时丢失或让危机提前到来。巧合不应连续为主角送来答案、权力、财富、爱情或胜利。

### 3. 建立状态账

每集结束时，至少检查这些维度：

- 目标：更接近、改变或失败；
- 关系：信任、边界、联盟或敌意改变；
- 资源：获得、消耗、暴露或失去；
- 能力：掌握、受限、反制或升级；
- 信息：谁知道什么，谁仍被蒙在鼓里；
- 地位：权力、声望、权限或归属改变；
- 风险：新风险出现，旧风险兑现或解除；
- 责任：人物承担、逃避或转移了什么。

### 4. 兑现必须改变局势

打脸、甜点、虐点、能力展示、真相揭示和关系确认都必须产生后果。旁人惊讶不是充分兑现；主角的处境、选择空间或下一步问题必须改变。

## 八、场景、台词与内心独白

### 1. 写可被画面和声音表达的内容

写行动、空间关系、可见反应、声音、对白和可被表演的停顿。避免用大段抽象心理解释代替戏剧行动。

可以写：

- 她把合同推回去，删掉签名页；
- 他听见那个称呼后停了一下，仍把门打开；
- 会议室里没人接话，负责人的手机先响了。

避免只写：

- 她百感交集；
- 他感到命运无常；
- 所有人都被强大气场震慑。

### 2. 只标关键表演状态

在选择、谎言、关系转向和信息揭示处标明表演状态。普通动作不需要每句配一个表情形容词。状态应具体，如“努力保持礼貌，但不再让步”，而非“情绪复杂”。

### 3. 台词执行规则

- 让人物用自己的词汇、节奏和回避方式说话；
- 让对白执行谈判、试探、攻击、防守、遮掩、请求或决定；
- 删除角色已经知道、只是说给观众听的说明；
- 允许沉默和动作承担信息；
- 使用短句或长句取决于人物、场景和表演，不设统一字数；
- 用试读检查是否可说、是否有呼吸空间；
- 将最终语速和镜头容纳量交给导演与声音工种核算。

### 4. 内心独白规则

仅在内心独白能提供下列价值时使用：

- 与表面行动形成反差；
- 揭示无法通过外部行为表达的抉择；
- 建立可靠或不可靠叙述；
- 提供特定形式所需的喜剧、悬疑或主观视角。

不要复述画面，不要依据 MBTI 决定长短，不要默认所有作品需要飘字 OS。

## 九、长剧、短剧与分季适配

### 1. 短剧模式

- 尽快让观看承诺、即时目标或异常可见；
- 提高有效事件密度，而不是提高争吵频率；
- 每集同时提供推进、局部兑现和下一步动力；
- 合并功能重复的角色、场景与支线；
- 保留必要的因果和情绪过渡，不用“快节奏”掩盖跳变。

### 2. 长剧模式

- 先确定全剧主线、阶段目标、人物弧和支线权限；
- 允许氛围、生活细节和关系沉淀，但每集仍有明确戏剧任务；
- 用阶段变化而非连续身份曝光维持动力；
- 重要反转提前埋因，避免临时补设定；
- 给配角弧和世界规则足够的兑现空间。

### 3. 分季模式

- 每季拥有独立目标、主要阻力、高潮选择和阶段闭环；
- 季终可以打开更大问题，但不能取消本季应有的兑现；
- 总季数留空表示不预设终点，不表示无限重复同一冲突；
- 单季集数留空时，根据当前季事件容量规划，不强套 5 集、6 集或 10 集；
- 当前季扩容或缩短时，优先保护关键选择和兑现，不用填充集补数量。

## 十、批次、连续性与项目档案

### 1. 批次可配置

批次大小服从用户审校方式、单集长度和项目风险。用户未指定时：

- 需要先验证口吻或结构时，交付较小样本；
- 结构已确认且连续性稳定时，增加批次；
- 不把“每批 5 集”写成固定规则。

### 2. 每批开始前执行三回锚

- **回锚结构**：当前批次在全剧，以及当前季（如适用）承担什么任务；
- **回锚连续性**：上一批结束时人物、关系、资源、信息和伏笔处于什么状态；
- **回锚规则**：当前版本故事引擎、形式约束和用户最新修改是什么。

### 3. 定稿后更新连续性

只在本批剧本确认或按用户要求视为定稿后更新。记录：

- 已发生且不可撤销的事件；
- 主要人物当前目标与情绪落点；
- 关系状态和边界；
- 资源、能力、伤势、身份与权限；
- 各角色已知信息；
- 已埋、已回收、延后和废弃的伏笔；
- 尚未兑现的承诺；
- 下一批必须承接的动作或问题。

用压缩骨架保存长期信息，用最近批次摘要保留短期细节。不要把完整聊天记录当连续性系统。

## 十一、形式适配与专业可信度

### 真人影视

控制场景、演员、群演、危险动作和特殊视觉要求；让复杂设定通过可见行动而非说明会呈现。

### 动态漫或动画

可以扩大世界和动作尺度，但仍需维持角色造型、空间方位、能力规则和动作因果的一致性。

### 小说或故事大纲

可以保留更多主观视角和叙述，但如果交付目标是后续影视化，仍需标明可转化的场景、行动和状态变化。

### 专业与历史题材

医疗、法律、金融、军工、刑侦、心理、工程和历史制度等内容，需要核实后再写成事实。无法核实时：

- 明确标注虚构化；
- 降低操作细节；
- 避免给出现实建议；
- 不用专业术语装饰明显错误的因果。

## 十二、内容安全与公开发布边界

- 不美化强制控制、跟踪、羞辱、家暴、性暴力、权力胁迫或资源封锁；
- 不把犯罪、黑恶势力、武器、侦查规避和伤害步骤写成可执行教程；
- 不让虚构神医、投资奇迹或法律桥段冒充现实建议；
- 不把性别、年龄、地域、民族、疾病、残障、贫穷或职业当作天然的道德判断；
- 不复制第三方作品的独特情节、长段台词、角色组合或风格指纹；
- 使用类型惯例时重新组合人物目标、规则、因果和主题；
- 不承诺爆款、播放量、付费率或平台推荐；
- 平台趋势和市场偏好只在用户明确要求时通过有日期的调研处理，不写进稳定规则。

## 十三、共同硬门槛

交付前逐项判断 PASS、REVISE 或 N/A。任一适用项为 REVISE 时，先修正文稿再交付。

1. **配置门槛**：长短剧、分季和本次范围已读取；留空参数没有被伪造为用户决定。
2. **承诺门槛**：主要观看承诺和当前阶段兑现清楚，且没有商业成功保证。
3. **主动性门槛**：主角至少做出一次推动本集或本阶段的主动选择。
4. **因果门槛**：核心转折能写成“信息或压力—行动—后果—下一选择”的链条。
5. **变化门槛**：每集至少一个故事状态真实改变，不以重复争吵充当推进。
6. **人物门槛**：主要人物的行为符合档案；转变有过程；配角不只是工具。
7. **兑现门槛**：局部兑现发生在新的前向问题之前，不用假悬念克扣已承诺信息。
8. **可拍门槛**：关键内容能由画面和声音表达，场景不是抽象心理说明。
9. **连续性门槛**：人物、时间、空间、资源、能力、信息和伏笔无未解释冲突。
10. **关系门槛**：恋爱权重可为 0；存在亲密关系时尊重同意、边界和责任。
11. **表述门槛**：没有把市场分类写成性别心理定律，没有用刻板印象代替人物。
12. **专业门槛**：敏感专业内容可信或明确虚构，不构成现实操作建议。
13. **安全门槛**：伤害、犯罪和权力滥用没有被无代价美化。
14. **交接门槛**：文学剧本与导演指令分开，连续性更新足够支持下一工种接手。
